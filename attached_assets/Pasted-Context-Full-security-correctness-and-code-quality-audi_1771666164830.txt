Context
Full security, correctness, and code-quality audit of the Tabibi Healthcare React Native / Express.js application. Issues are grouped by severity and ordered for remediation priority. Each issue includes the affected file, line, description, and recommended fix.

CRITICAL — Fix Immediately
C1. Unauthenticated Sensitive API Endpoints
Files: server/routes/avicenna.ts lines ~153, ~167
Two endpoints expose aggregate health data without auth:

GET /api/avicenna/trending — trending conditions/symptoms/medicines
GET /api/avicenna/knowledge/:category — Iraqi health knowledge base

Fix: Add requireAuth to both route handlers.

C2. Prompt Injection via Unsanitized Profile Data
File: server/routes/ai.ts lines ~205–236
Patient profile fields (name, conditions, medications, allergies) and the full Avicenna context string are directly string-concatenated into the AI system prompt. sanitizeInput() runs only on the user's chat message, not on profile data. An attacker can store a malicious instruction (e.g., in a medication name) via a prior API call, which is then injected into every subsequent assessment.
Fix: Sanitize all profile fields before injection. Use structured JSON delimiters (not free-form strings) to separate data from instructions. Apply sanitizeInput() to every field in systemContext.

C3. Encryption Key Falls Back to DATABASE_URL
File: server/encryption.ts lines ~7–16
When ENCRYPTION_KEY env var is missing, the key is derived from DATABASE_URL via SHA-256. DATABASE_URL frequently appears in logs, CI configs, and .env files — an attacker with read access can decrypt all PII (patient names, addresses, medications, orders).
Fix: Remove the DATABASE_URL fallback. Throw at startup if ENCRYPTION_KEY is not set. Document this requirement.

C4. System Prompt Explicitly Bans Medical Disclaimers
File: server/routes/ai.ts line ~83
The prompt instructs the AI: "Do NOT add medical disclaimers, caveats, or 'consult a doctor' reminders." If the client-side safety messaging fails to render, users receive clinical recommendations with zero safety warnings — a direct patient safety risk.
Fix: Remove this instruction. Include a mandatory, server-generated safety footer in every SSE response before streaming ends.

C5. Emergency Alert Parsed via Regex Without Schema Validation
File: app/assessment.tsx lines ~320–342
Emergency alerts are detected by matching a free-form regex against the LLM's raw text stream. There is no schema validation on the parsed object; any JSON matching {"emergency": true ...} triggers the emergency overlay, including AI hallucinations. No whitelist of approved emergency conditions exists.
Fix: Define a strict Zod schema for emergency objects (required condition, action fields; condition must be in an approved list). Validate before calling setEmergency(). Log all triggers server-side.

C6. Race Condition — Token Refresh Without Synchronization
File: app/assessment.tsx lines ~244–259
On a 401 response, the code fires a second fetch with refreshed headers without preventing concurrent refreshes or ensuring the new token was actually written before the retry.
Fix: Use a refresh-lock pattern (a Promise that all concurrent requests await). Move token refresh to lib/query-client.ts as a shared interceptor.

HIGH — Fix Within 1 Week
H1. Silent Decryption Fallback Returns Plaintext
File: server/encryption.ts line ~32
decrypt() returns the input unchanged when the format is invalid (not 3 colon-separated hex parts). Corrupted or un-encrypted data silently passes as plaintext, making tampering undetectable.
Fix: Throw an error on malformed input. Log all decryption failures.

H2. No Per-User Rate Limiting on AI Endpoints
File: server/index.ts / server/routes/ai.ts
Rate limiting is IP-based. An authenticated user behind a VPN or shared IP is effectively unlimited, while legitimate users on the same IP get blocked. AI quota abuse is possible.
Fix: Implement per-userId rate limiting using a map or Redis store for AI endpoints (/api/assess, /api/analyze-medication, /api/check-interactions).

H3. Missing Auth on Geolocation Endpoints
File: server/routes/geo.ts lines ~72–212
All geo endpoints (/api/nearby-facilities, /api/place-photo/:photoRef, /api/place-details/:placeId) are publicly accessible. This enables Google Maps API quota exhaustion and facility enumeration.
Fix: Add requireAuth to all geo routes, or at minimum add per-IP strict rate limiting with CAPTCHA for unauthenticated access.

H4. IDOR on Photo References
File: server/routes/geo.ts line ~194
photoRef path param is passed directly to Google Maps with no validation. Attackers can probe arbitrary photo references.
Fix: Validate photoRef matches an allowlist generated from recent legitimate searches, or require a signed/expiring token for photo access.

H5. enrichProfileFromAssessment() Fragile Array Mutation
File: server/avicenna.ts lines ~118–159
Conditions/medications/allergies arrays are mutated by decrypting, pushing, and re-encrypting — but includes() checks run on already-decrypted data that may not match due to formatting differences. If decryption fails silently (see H1), the returned null is spread-merged, wiping data.
Fix: Wrap the entire flow in a try/catch. Normalize strings before deduplication. Use a Set for duplicate detection. Handle null returns explicitly.

H6. Race Condition in Population Analytics Counter
File: server/avicenna.ts lines ~85–116
Count increment is a read-modify-write: count: existing.count + 1. Two concurrent requests read the same value and each increment it, losing one write.
Fix: Use a SQL atomic increment: .set({ count: sql\count + 1` }). Add a unique constraint on (category, itemName, periodDate, region)` and use upsert.

H7. Geo Coordinate Injection
File: server/routes/geo.ts lines ~106–125
parseFloat() accepts strings like "52 DROP TABLE" (returns 52) and values outside valid coordinate ranges.
Fix: Validate lat ∈ [-90, 90] and lng ∈ [-180, 180] with explicit range checks after parsing.

H8. rPPG RGB Values Not Range-Validated
File: server/routes/rppg.ts / server/rppg-worker.js
Schema uses z.number() with no bounds; worker checks >= 0 only. Allows Infinity, NaN, and values outside [0,255].
Fix: Change schema to z.number().min(0).max(255). Add isFinite() checks in the worker. Raise SNR threshold from 0.08 to 0.15.

H9. Iraqi Brand Recommendations Without Market Validation
File: server/routes/ai.ts lines ~96–108 (system prompt)
The prompt instructs the AI to recommend specific Iraqi brand names and prices, but brands are not validated against a live or periodically updated inventory. The AI may hallucinate brands, doses, or prices.
Fix: Strip brand recommendation instructions from the open prompt. Supply only validated brands from iraqiHealthKnowledge DB as a structured allowlist in the prompt context.

H10. Memory Leak — useEffect Missing Dependencies / No Cleanup
File: app/assessment.tsx lines ~482–487; app/(tabs)/orders.tsx lines ~40–64

useEffect in assessment references sendMessage without including it in deps, causing stale closures.
loadOrders() sets state after unmount (no AbortController cleanup).

Fix: Add all referenced values to dependency arrays. Use AbortController for fetch calls inside useFocusEffect/useEffect and abort on cleanup.

MEDIUM — Fix Within 2 Weeks
M1. Order Status Transitions Not Validated
File: server/routes/orders.ts lines ~115–119
Any status string can be patched to any other status (e.g., cancelled → confirmed).
Fix: Enforce a state machine: pending → confirmed | cancelled, confirmed → delivered | cancelled. Reject invalid transitions with 400.

M2. Error Messages Leak in Non-Production
File: server/index.ts line ~254
Non-500 errors return raw error.message, which can include Drizzle query details, column names, and stack traces.
Fix: In production (NODE_ENV === 'production'), return only status code and a generic message. Log details server-side.

M3. sanitizeInput() Regex Bypass
File: server/routes/ai.ts lines ~15–36
Current blocklist approach (removing strings like "ignore previous") is bypassable via unicode, spacing, or fragmentation. User messages only, not profile data (see C2).
Fix: Complement with a length-limited whitelist of allowed character sets for medical input. Limit message content to 2,000 characters.

M4. Stale Closures in AvicennaContext and Home Screen
File: contexts/AvicennaContext.tsx line ~155; app/(tabs)/index.tsx lines ~283–299
useMemo dependency arrays omit callback functions, causing stale references after re-renders.
Fix: Include all callback references in dependency arrays or stabilize with useCallback.

M5. Global retry: false in QueryClient
File: lib/query-client.ts lines ~110–123
All queries fail immediately on transient network errors — no retry on flaky mobile connections.
Fix: Set retry: 2 with exponential backoff. Exclude 4xx errors from retries.

M6. No CSRF Protection
File: server/index.ts
POST/PATCH endpoints rely solely on Bearer tokens. API could be abused via cross-site form submissions in WebView contexts.
Fix: Implement csurf or double-submit cookie for state-changing operations, or add a custom header (X-Requested-With) check.

M7. decrypt() Silent Failure in avicenna.ts
File: server/avicenna.ts lines ~20–31
decryptHealthData() catches all errors and returns null. Tampered or corrupted records silently vanish.
Fix: Log the error with user ID and field name. Surface a non-null sentinel so callers can distinguish "empty" from "corrupted".

M8. Seed Data Lacks Citations, Dates, and Completeness
File: server/seed-iraqi-knowledge.ts
Epidemiological prevalence rankings have no lastUpdated timestamp. Drug prices (e.g., ~2,000-5,000 IQD) will become stale. Treatment protocols (brucellosis, heat stroke) are incomplete.
Fix: Add lastUpdated, citationSource, and clinicalReviewDate fields. Mark prices as approximate with embedded date. Flag incomplete protocols for clinical review.

M9. Assessment Results Saved Without Schema Validation
File: app/assessment.tsx lines ~390–407
Parsed assessment objects are persisted to local storage without validating required fields (condition, recommendations, medicines).
Fix: Define a Zod schema for Assessment and validate before saving. Reject and log partial results.

M10. Geo API Exposes Internal Error Status to Client
File: server/routes/geo.ts lines ~113–124
Returns Google Places API error: REQUEST_DENIED directly to the client.
Fix: Log the full error internally; return a generic "Facility search unavailable" to the client.
